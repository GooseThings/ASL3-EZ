Configuration Guide

AMI requires credentials set in the systemd service file at /etc/systemd/system/ASL3-EZ.service.

Step 1: Ensure manager.conf has a user with command write access:

[general]
enabled = yes
port = 5038
bindaddr = 127.0.0.1

[admin]
secret = your_secret_here
read = system,call,log,verbose,command,agent,user,config,dtmf,reporting,cdr,dialplan
write = system,call,log,verbose,command,agent,user,config,dtmf,reporting,cdr,dialplan
permit = 127.0.0.1/255.255.255.0

Step 2: Set matching credentials in the service file:

Environment="AMI_USER=admin"
Environment="AMI_SECRET=your_secret_here"

Step 3: Reload and restart:

sudo systemctl daemon-reload
sudo systemctl restart ASL3-EZ
